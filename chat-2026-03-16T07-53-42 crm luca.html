3. API backend per gestire le zone

In main.ts ho aggiunto tre endpoint REST, tutti protetti da JWT:

  1. GET /api/agent-zones

    • Autenticazione obbligatoria.
    • Se chi chiama è AGENT o altro non-admin:
      • vede solo le proprie zone, filtrate per auth.agencyId e auth.id.
    • Se chi chiama è ADMIN (SUPER_ADMIN / AGENCY_ADMIN):
      • può filtrare per agentId (query string).
      • per AGENCY_ADMIN si restringe comunque all’agenzia corrente.
    • Ritorna:
    JSON

  2. POST /api/agent-zones

    • Solo per ruoli SUPER_ADMIN o AGENCY_ADMIN (controllo con isAdminRole).

    • Body richiesto:

      JSON

    • Se mancano agentId, city o zone → 400 “Missing required fields”.

    • agencyId viene preso:

      • da auth.agencyId se presente, altrimenti
      • dall’agente indicato in agentId.

    • Crea il record AgentZone e restituisce:

      JSON

  3. DELETE /api/agent-zones/:id

    • Solo SUPER_ADMIN / AGENCY_ADMIN.
    • Controllo che la zona esista e appartenga alla stessa agenzia dell’admin (se non SUPER_ADMIN).
    • Elimina la zona e ritorna { "success": true }.

Quindi solo l’admin (o agency admin) può creare e cancellare assegnazioni di zona. Gli agenti eventualmente potranno in futuro solo consultarle, non modificarle.
